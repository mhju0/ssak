# Ssak App (Plan 3) — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development (logic tasks) or executing-plans. Steps use checkbox (`- [ ]`). **Two verification modes:** pure logic (`GardenModel`) is TDD'd with `swift test` and injected time; screens are *authored against the headless render→PNG→inspect loop* (reuse `SsakArt.pngData`). Only the live share sheet, touch gestures, and `scenePhase` need a real simulator — flagged per task.

**Goal:** Wire `SsakCore` (engine) and `SsakArt` (art) into the playable iOS game: windowsill home, shelf collection, watering, sharing, onboarding, and reconcile-on-open — as a testable `SsakApp` SwiftPM library plus a thin iOS shell.

**Architecture:** A new SwiftPM library `SsakApp` (iOS 16 / macOS 13) depends on `SsakCore` + `SsakArt`. All game logic lives in **`GardenModel`** — a pure, time-injected reducer over `GameState` (load/new, reconcile-on-open, water, collect-on-bloom, replant, derived flags) so it's unit-testable with no UI. SwiftUI screens (`WindowsillView`, `ShelfView`, `DropGauge`, `OnboardingView`, `BloomCard`, `RootView`) compose `SsakArt.PlantView` and are render-verified headless via a `SsakAppRender` macOS executable (same `pngData` trick as the art). A thin **iOS app shell** (`App/` + XcodeGen `project.yml`) hosts `RootView`; it is the only Xcode-specific piece.

**Tech Stack:** SwiftUI, `ImageRenderer`, `ShareLink`/`UIActivityViewController`, SwiftPM, XcodeGen (project generation, no GUI).

## Global Constraints

- New package dir `SsakApp/` (sibling to `SsakCore/`, `SsakArt/`), depends on both via local path. Library target `SsakApp` platforms `.iOS(.v16), .macOS(.v13)`; macOS executable `SsakAppRender`; test target `SsakAppTests`. — verbatim.
- **All game logic in `GardenModel` is pure + time-injected** (every method takes `now: Date` / `calendar`); no `Date()`/`Date.now`/`Calendar.current` in logic paths. Views may read the clock only at the interaction boundary. — verbatim.
- Reuse `SsakCore` (`GrowthEngine`, `PlantState`, `GameState`, `PlantStore`, `SpeciesCatalog`, `GrowthStage`, `GrowthTuning`) and `SsakArt` (`PlantView`, `pngData`, `SpeciesPalette`) — do not reimplement. Marigold is the starter (`SpeciesCatalog.starter`).
- **Forgiving realism (spec §3.1, D-ledger):** never permadeath, shelf never lost, wilt only recoverable, over-water warns but doesn't punish. — verbatim.
- **Streak "reset on missed day" is derived at the UI** from `lastWateredAt` (spec §3.2 deferral, noted at `GrowthEngine.reconcile`) — `GardenModel.isStreakAlive(now:)`, not an engine change.
- iOS shell verified on a real Simulator/device by the user; the library, logic, and screen renders are verified headless here. MIT, © 2026 Michael Ju.

### Task 2: GardenModel — the pure reducer (TDD)

**Files:** Create `SsakApp/Sources/SsakApp/GardenModel.swift`; Test `SsakApp/Tests/SsakAppTests/GardenModelTests.swift`.

**Interfaces (produce exactly):**
```swift
@MainActor final class GardenModel: ObservableObject {
    @Published private(set) var state: GameState
    let store: PlantStore
    var tuning: GrowthTuning = .default

    init(store: PlantStore, calendar: Calendar = .current)   // loads store.load() or starts a new marigold at `now` (pass now via start(now:) — see below)

    // pure, time-injected transitions (each persists via store.save):
    func startNewIfEmpty(now: Date)                          // if no save, plant SpeciesCatalog.starter at now
    func reconcileOnOpen(now: Date)                          // GrowthEngine.reconcile(state.plant, to: now, ...) → save
    func water(now: Date)                                    // GrowthEngine.water(...) → save
    func pressAndReplant(_ next: Species, now: Date)         // if current is .bloom: add speciesID to collected (once), plant `next`, save

    // derived (no mutation):
    var species: Species                                     // SpeciesCatalog.species(id: state.plant.speciesID)
    var stage: GrowthStage                                   // GrowthEngine.stage(forProgress: state.plant.progress)
    var moistureFraction: Double                             // state.plant.moisture / tuning.moistureMax, clamped 0...1
    func hasWateredToday(now: Date) -> Bool                  // GrowthEngine.hasWateredToday(state.plant, now:)
    func isStreakAlive(now: Date, calendar: Calendar) -> Bool// false if a full day passed since lastWateredAt (spec §3.2 at UI)
    func wouldOverwater(now: Date) -> Bool                   // already watered today AND moisture > tooWetThreshold
    var isNursing: Bool                                      // state.plant.isNursing
    var collected: [String]                                  // state.collected
    var isGardenComplete: Bool                               // Set(collected) ⊇ all six species ids
}
```

- [ ] Step 1: Write failing tests (discriminating, injected time): (a) `startNewIfEmpty` on empty store plants marigold, progress 0, stage seed; (b) `reconcileOnOpen` after N days of a watered/healthy plant advances progress (assert stage changed); (c) `water` raises moisture and sets watered-today true; (d) `wouldOverwater` true only when watered-today AND moisture>tooWet; (e) `isStreakAlive` false after ≥2 day gap, true same-day; (f) reaching `.bloom` then `pressAndReplant` adds species to `collected` exactly once and resets to the new seed; (g) `isGardenComplete` true only with all six ids. Each must fail before implementation.
- [ ] Step 2: Run → FAIL. `cd SsakApp && swift test --filter GardenModelTests`.
- [ ] Step 3: Implement `GardenModel` delegating to `GrowthEngine` + `PlantStore`; `isStreakAlive` = `calendar.dateComponents([.day], from: startOfDay(lastWateredAt), to: startOfDay(now)).day ?? 99 < 1` (alive only if watered today or… choose: streak alive if no *full* missed day, i.e. delta ≤ 1 and watered yesterday/today — pin exact rule in a test).
- [ ] Step 4: Run → PASS. Full `swift test` green.
- [ ] Step 5: Commit `feat(app): GardenModel pure reducer over engine + persistence`.
